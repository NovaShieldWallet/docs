**SOL ONLY** - Send SOL with complete privacy and automatic refund protection.

## Auto-Refund Protection

If transfer fails, your SOL is **automatically refunded** to your refund address (minus network fees). No manual intervention needed.

## How It Works

1. Call endpoint with amount, recipient, and optional refund address
2. Receive temp wallet address
3. Send SOL to temp wallet (you keep your private key!)
4. Transfer auto-executes when funded (checks every 2s)
5. If fails: Auto-refunds to your address
6. Temp wallet expires in 10 minutes

## Request

```json
{
  "amount": 0.1,
  "toAddress": "RECIPIENT_ADDRESS",
  "network": "mainnet",
  "refundAddress": "YOUR_REFUND_ADDRESS"
}
```

**Required:** `amount`, `toAddress`  
**Optional:** `network` (default: devnet), `refundAddress` (recommended)

## Response

```json
{
  "success": true,
  "data": {
    "transferId": "transfer-123",
    "tempWallet": "TEMP_WALLET_ADDRESS",
    "amount": 0.1,
    "toAddress": "RECIPIENT_ADDRESS",
    "refundAddress": "YOUR_REFUND_ADDRESS",
    "expiresAt": "2024-01-01T12:00:00Z",
    "network": "mainnet"
  },
  "message": "Send SOL to tempWallet. Auto-executes when funded. Auto-refunds on failure."
}
```

## Next Steps

1. Send SOL to `tempWallet` address
2. Check status with `transferId`
3. Transfer completes automatically OR refunds on failure

## Details

- **Minimum**: 0.01 SOL
- **Expiry**: 10 minutes
- **Auto-refund**: Yes (minus network fees)
- **SOL ONLY**: No tokens

## Important

✅ **Auto-refund on failure** - Your funds are protected  
✅ **No private key needed** - You stay in control  
⚠️ **10 minute expiry** - Send SOL within this time  
⚠️ **SOL ONLY** - Do not send tokens
