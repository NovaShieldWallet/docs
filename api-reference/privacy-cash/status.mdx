---
title: 'Check Transfer Status'
openapi: 'GET /privacy-cash/status/{transferId}'
---

Check the status of a privacy cash transfer with detailed refund information.

## Parameters

- **transferId** (path, required) - Transfer ID from `/privacy-cash/send`

## Response

```json
{
  "success": true,
  "data": {
    "transferId": "transfer-123",
    "status": "completed",
    "tempWallet": "TEMP_WALLET_ADDRESS",
    "amount": 0.1,
    "recipient": "RECIPIENT_ADDRESS",
    "signature": "5J7x...",
    "refundAddress": "YOUR_REFUND_ADDRESS",
    "expiresAt": "2024-01-01T12:00:00Z",
    "createdAt": "2024-01-01T11:50:00Z",
    "completedAt": "2024-01-01T11:51:30Z"
  }
}
```

## Status Values

| Status | Description |
| ------ | ----------- |
| pending | Temp wallet created, awaiting SOL |
| funded | SOL received, processing transfer |
| completed | Transfer successful |
| failed-refunded | Transfer failed, SOL automatically refunded |
| failed | Transfer failed, refund also failed |
| expired | 10 minute timeout, funds returned |

## Refund Information

When `status` is `failed-refunded`, response includes:

```json
{
  "refunded": true,
  "refundSignature": "ABC123...",
  "refundAmount": 0.099,
  "refundAddress": "YOUR_REFUND_ADDRESS",
  "error": "Recipient address invalid"
}
```

## Transfer Flow

1. **pending** → Send SOL to temp wallet
2. **funded** → Processing transfer
3. **completed** → Done! OR **failed-refunded** → Auto-refunded
