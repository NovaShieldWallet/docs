## Overview

The health check endpoint verifies that the NovaShield Security API is operational and provides system status information.

## Use Cases

- **Service monitoring** - Check API availability
- **Integration testing** - Verify API connectivity
- **Status dashboards** - Display service health
- **Uptime monitoring** - Automated health checks

## Response

The endpoint returns a simple JSON response with:

- **status** - Service status (typically "healthy")
- **version** - Current API version (2.0.0)
- **mode** - Detection mode ("local-first")

## Local-First Detection

NovaShield uses a "local-first" approach, meaning:
- ✅ Primary analysis runs on our infrastructure
- ✅ Fast response times (typically <500ms)
- ✅ No external API dependencies for core checks
- ✅ Reliable and consistent results

## Status Values

| Status | Meaning |
|--------|---------|
| `healthy` | All systems operational |
| `degraded` | Some features may be slow |
| `maintenance` | Scheduled maintenance |
| `error` | Service experiencing issues |

## Best Practices

- Check this endpoint before making other API calls
- Implement retry logic for failed health checks
- Cache health status for 10-30 seconds to avoid excessive calls
- Monitor for version changes to stay updated

## No Authentication Required

The health check endpoint does not require any authentication and can be called freely for monitoring purposes.
